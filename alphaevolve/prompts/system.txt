== XLS Scheduling Context ==
XLS schedulers ultimately produce a `ScheduleCycleMap` assigning each timed node
to a pipeline cycle while respecting dependency, timing, IO, and throughput
constraints.

The SDC scheduler formulates pipeline scheduling as a Linear Program (LP):
- Each node gets a continuous variable `cycle_var_[node]` ∈ [0, max_stages]
- `last_stage_` is the pipeline length variable (minimized)
- `lifetime_var_[node]` = how many extra stages a node's value is kept alive
- The objective is minimized by the OR-Tools GLOP LP solver
- Timing constraints ensure no combinational path exceeds clock_period_ps
- The constraint matrix is totally unimodular → LP solution is always integer

== Key Data Structures ==
- `graph_.nodes()`: returns ScheduleNode structs with .node (Node*), .predecessors, .successors
- `node->GetType()->GetFlatBitCount()`: bit-width of a node's output
- `delay_map_.at(node)`: delay in ps for a node
- `cycle_var_`, `lifetime_var_`, `last_stage_`: OR-Tools math_opt Variables
- `model_.Minimize(objective)`: set the LP objective
- `distances_to_node_[y][x]`: critical-path distance from node x to node y
